# v0.14.0 SDK実装計画

## 方針

BRIDGEはAPIサーバーを直接提供しない。利用者はPythonまたはTypeScript SDKを組み込んで任意のサーバーアプリケーションを構築する。
SDKは探索を再実装せず、同梱されたBRIDGE CLIを子プロセスとして利用する。

## 実装項目

1. Python SDKとTypeScript SDKに同一の`BridgeClient.route()`契約を提供する。
2. Linux amd64/arm64、Windows amd64、macOS amd64/arm64の静的バイナリをSDKへ含める。
3. バイナリ解決順を、明示パス、`BRIDGE_BINARY`、同梱版、PATHの順に固定する。
4. SDKと同梱バイナリを同じ0.14.0として管理し、`bridge version`で検査する。
5. stdinへRoute Request JSON、stdoutからRoute Result JSON、stderrからwarning/errorを受け取る。
6. timeout、Pythonのcancel相当、TypeScriptのAbortSignal、子プロセス回収を実装する。
7. CLI終了コードをSDK例外へ変換する。
8. SDKは暗黙にファイル、Trace、Scenarioを生成しない。
9. 単体テストと実バイナリ統合テストを追加する。
10. バイナリ自動ダウンロード、自動更新、APIサーバーを実装しない。

## 完了条件

- Go、Python、TypeScriptの全テストが成功する。
- 両SDKから同じRequestを実行して同じ主要Resultを得る。
- 同梱バイナリだけで実行できる。
- 外部通信なしでインストール後に利用できる。
- timeout/cancel後に子プロセスが残らない。
- 自動ダウンロードを行うコード・文書が存在しない。
