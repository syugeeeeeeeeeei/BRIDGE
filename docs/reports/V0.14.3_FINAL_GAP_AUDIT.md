# v0.14.3 最終抜け漏れ監査

## 監査範囲
Work契約、ANCHOR・BOLTS・TRUSSの計上、Budget Ledger、profile Action Event、TRAFFIC artifact、HEALTHYの正当性・Work・比較・回帰判定、schema、用語集、CLIを対象とした。

## 発見し修正した漏れ
1. TRUSS Budget Ledgerが公開artifactへ保存されていなかった。
2. HEALTHYがtrace manifest／JSONLを自動読込みしていなかった。
3. Bidirectional DijkstraとReachabilityがprofile Action Eventを発行していなかった。
4. Reachabilityが有効pathに対してdistance=0を返していた。
5. Reconstructed WorkとReported Workの比較にWorkerCountが混入していた。
6. source artifactのdigestとHealth Check生成時刻が結果契約に不足していた。

## 最終判定
- path、distance、Found、ExactはHEALTHYが独立検証できる。
- Workは保存則、Budget Ledger、完全profile traceの3段階で検証できる。
- 全現行solverで固定fixtureおよびprofile E2EのReported/Reconstructed Workが一致する。
- sampling、drop、truncation、digest不一致があるtraceは検証済みとして扱われない。
- invalid Runは性能比較から除外される。
- 未実装アブレーションは黙って無視されない。
- Workの正本は`WorkMetrics`のみである。旧fieldは互換表示用の派生値であり、独立した評価入力には使用しない。

必須級の未接続field、未検証の現行solver経路、二重のWork正本は確認されなかった。Work計上検証・HEALTHY新設計画は完了とみなす。
