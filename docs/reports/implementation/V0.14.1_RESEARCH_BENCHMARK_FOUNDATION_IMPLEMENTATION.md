# v0.14.1 研究ベンチマーク基盤 実装報告

## 実装範囲

本変更では、計画の依存順に従い、Phase 1を実装し、Phase 3のObservation Mode公開契約を先行整理した。

### Phase 1

- Scenarioへ`warmup_runs`、`randomize_order`、複数`queries`を追加した。
- 旧`endpoints`を`query_id=default`として正規化し、既存Scenario互換を維持した。
- Runを`scenario_id × algorithm × graph_instance_id × query_id × repetition`で識別した。
- warm-upをraw Runとして保存し、summaryとacceptanceから除外した。
- graph、query、quality、environment、output metadataを研究結果へ追加した。
- raw observationsをトップレベル`raw_runs`へ保持した。
- Work、solver time、end-to-end timeへmean、sample standard deviation、min、p50、p95、max、95% confidence intervalを追加した。
- deterministic randomizationをScenario seed集合から導出した。

### Observation Mode整理

- 公開modeを`off`、`summary`、`trace`、`profile`へ固定した。
- 旧`metrics`、`debug`をvalidationおよびGATE公開境界から除外した。
- ULTRASOUND CollectorとRecorderを新modeへ同期した。

## 責務境界

- Run展開、metadata計算、統計集計はTRAFFICに限定した。
- GATEはmode validationと公開結果整形だけを行う。
- ULTRASOUNDはmode別event選別を行う。
- solver private stateへのTRAFFIC依存は追加していない。
- TRUSS、ANCHOR、BOLTSの探索制御は変更していない。
