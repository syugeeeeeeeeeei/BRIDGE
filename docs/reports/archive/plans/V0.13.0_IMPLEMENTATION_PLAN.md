# v0.13.0 実装計画

## 目的

ULTRASOUNDの観測生成と保存先を分離し、routeとbenchmarkから暗黙のファイル生成なしで利用できるようにする。

## 実装項目

1. Collectorへsequence、elapsed、delta、mode選別、truncation、集計を集約する。
2. Discard、Memory、Writer、File、Callback、Multi Sinkを共通EventSinkとして提供する。
3. routeへObservation Resultと明示的な`--trace-output`を追加する。
4. benchmarkへ明示的な`--trace-dir`を追加し、run単位のJSONLへ分離する。
5. Observationの有無でpath、distance、work、exact、solverが変化しないことをテストする。
6. Trace JSONLでNaNおよびInfinityを安定した文字列表現へ正規化する。
7. I/O失敗を終了コード3、timeout/cancelを終了コード4として維持する。
8. 公開SchemaとCLIドキュメントを更新する。

## 完了条件

- `go test ./...`と`go vet ./...`が成功する。
- routeは明示指定時だけTraceファイルを生成する。
- benchmarkは明示指定時だけrun単位Traceを生成する。
- Observation OFFとTRACEで探索結果およびWorkが一致する。
- Trace EventがJSONLとして最後まで書き出せる。
