# BRIDGE v0.15.2 実装報告

## 目的

v0.15.1で回収した探索正当性を前提に、Benchmark Artifactの完全性とSDKが保持する公開エラー情報を強化した。

## 実装

- Benchmark ZIP作成直前にExecution Manifestを確定する。
- `manifest.json`へ、Manifest自身を除く全成果物の相対パス、ファイルサイズ、SHA-256を記録する。
- `artifact validate`で必須ファイル、未記載ファイル、欠損、重複、サイズ、SHA-256、Schema、Suite ID、Execution ID、Run件数を検査する。
- `artifact evaluate`は完全性検証に成功したArtifactだけを評価する。
- PythonおよびTypeScript SDKの例外へ、`code`、`category`、`retryable`、`request_id`を保持する。
- ApplicationおよびSDK package versionを0.15.2へ更新する。

## 完全性境界

`manifest.json`は自分自身のDigestを保持しない。生成予定のZIPもManifest対象外とする。その他の通常ファイルは全てManifest記載を必須とする。
