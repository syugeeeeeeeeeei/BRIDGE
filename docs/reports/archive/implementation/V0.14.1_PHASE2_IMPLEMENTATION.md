# v0.14.1 Phase 2 実装記録

## 実装済み

- COREに`TimeBreakdown`と`SystemMetrics`の型付き契約を追加した。
- TRUSSでANCHOR、BOLTS、fallback、Supervisor、Arbiter、orchestrationの時間を区別した。
- GATE公開結果へ時間内訳を追加し、GATE境界の追加時間を`gate_ms`として記録した。
- TRAFFICのraw runへ時間内訳、割当byte、malloc回数、GC回数、heap前後値を保存した。
- runtime計測および観測I/Oを探索Workへ含めない。

## 制約

`heap_alloc_boundary_max`はrun前後の`HeapAlloc`最大値である。連続サンプリングやruntime profileによる厳密な瞬間最大値はPhase 3の`profile` modeで扱う。
