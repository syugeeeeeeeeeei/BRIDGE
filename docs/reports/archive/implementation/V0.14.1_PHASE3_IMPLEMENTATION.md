# v0.14.1 Phase 3 実装記録

## 実装範囲
- BEARING canonical event vocabulary と Event Class
- ULTRASOUND `off/summary/trace/profile` の実動作分離
- summaryの非I/O集計、trace/profileのJSONL保存
- observer処理時間とsink書込み時間の記録
- TRUSS fallback/certification event
- replayのquality history、budget history、fallback/certification count
- observation mode非干渉テスト

## mode契約
- `off`: observerを接続しない。
- `summary`: control/candidate eventのみ集計し、event streamを保存しない。
- `trace`: control/candidate/detail eventを保存し、探索状態と品質推移をreplay可能にする。
- `profile`: traceに加えて高頻度Action eventと観測コストを保存する。

## 非干渉
modeはevent生成量と保存量のみを変更し、path、distance、found、exact、Work、Step、tie-breaking、seed消費順へ影響してはならない。
